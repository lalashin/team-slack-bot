## 1. 에이전트가 구현·수정한 내용

### 프로젝트 구조
- `src/index.js` — 봇 진입점 (`app.start`, 포트 로그)
- `src/app.js` — `@slack/bolt` `App` 생성, `registerHandlers` 연결
- `src/config/slack.js` — `SLACK_BOT_TOKEN`, `SLACK_SIGNING_SECRET`, `PORT`, `DB_PATH` 등
- `src/handlers/index.js` — 슬래시 커맨드·이벤트 등록
- `src/handlers/commands.js` — `/todo`, `/list`, `/status`, `/help`
- `src/handlers/events.js` — `app_mention`, DM용 `message` 처리
- `src/services/dataService.js`, `src/services/taskStore.js` — 인메모리 작업 저장 (재시작 시 초기화)
- `src/utils/validators.js` — 작업 제목 검증
- `src/db/init.js` — SQLite 연동용 스텁 (Phase 4 예정)
- `src/messages/help.js` — 도움말 블록 한글 통합

### 동작·설정
- **Socket Mode**: `.env`에 `SLACK_APP_TOKEN`(`xapp-`)이 있으면 자동으로 Socket Mode (로컬에서 ngrok 없이 테스트 가능). 없으면 HTTP 모드.
- **디버그**: `LOG_SLACK_EVENTS=1` 시 수신 이벤트 타입 로그
- **멘션/DM**: DM은 `app_mention`이 오지 않으므로 `message` + `message.im` 구독 전제. 멘션 본문에 따라 한글 안내·`도움`/`help` 시 도움말 블록 표시 (동일 영문만 반복하던 동작 개선).

### 의존성
- `package.json` 기준: `@slack/bolt`, `express`, `sqlite3`, `dotenv` 및 devDependencies 설치 완료 (`npm install`).

### Git (에이전트가 수행한 이력)
- 원격 `https://github.com/lalashin/team-slack-bot.git`에 초기 푸시, 커밋 메시지 `chore: initial project setup`으로 amend 후 force-push (당시 문서 위주 스캐폴드).

## 2. 사용자가 세팅한 부분 (대화·스크린샷 기준)

- **Slack 앱** ([api.slack.com/apps](https://api.slack.com/apps)): `Team Slack bot` 생성, manifest 또는 수동 설정
- **OAuth (Bot Token Scopes)**: 예) `chat:write`, `commands`, `app_mentions:read`, `reactions:read`, `reminders:read`, DM용 `im:history` 등
- **토큰·시크릿**: `.env`에 `SLACK_BOT_TOKEN`(`xoxb-`), `SLACK_SIGNING_SECRET` — (토큰은 채팅에 노출 시 재발급 권장)
- **Socket Mode** 켜기 + **App-Level Token** (`connections:write`) → `SLACK_APP_TOKEN`
- **Event Subscriptions (Bot Events)**: `app_mention`, DM용 `message.im` 등 — 채널 멘션 수신에 `app_mention` 필수
- **워크스페이스에 앱 설치(Reinstall)** 및 **채널에 봇 초대** (`/invite @Team Slack bot`)
- **Slash Commands**: `/todo`, `/list`, `/status`, `/help`를 Slack 앱 설정에 등록 (일반 메시지의 `/help`와 구분)

## 3. 테스트 결과 (정리)

| 항목 | 결과 |
|------|------|
| `npm run dev` | `Team Slack Bot listening…` 및 Socket Mode 시 `Now connected to Slack` 확인 |
| 채널 멘션 | 초기에는 `app_mention` 미구독 등으로 무응답 → 구독·재설치 후 응답 확인 흐름 |
| DM | `app_mention` 미발생 → `message.im` + 코드상 DM 핸들러로 대응 |
| 슬래시 `/help` | 앱에 커맨드 미등록 시 Slack이 “유효하지 않은 명령” 안내 — 등록 후 입력창 맨 앞 `/help` 권장 |
| 멘션만 반복 영어 | 코드가 본문 미처리였음 → 한글 안내·`도움` 처리로 개선 |

## 4. 참고

- 작업 데이터는 현재 **인메모리**라 서버 재시작 시 사라짐. 영구 저장은 설계대로 SQLite Phase에서 연동 예정.
- 보안: 봇 토큰·시크릿은 `.env`만 사용하고 저장소·채팅에 노출하지 않기.
