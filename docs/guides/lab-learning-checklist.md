# 이 실습에서 배우는 것 (체크리스트)

**한 페이지 요약** — Claude + Cursor로 **문서 기반 설계 → 구현 → Slack 연동**까지 경험할 때, 아래 항목을 스스로 설명할 수 있으면 실습 목표 달성에 가깝습니다.

---

## 도구·역할

- [ ] **Cursor**에서 프로젝트 열기, 파일 탐색, 에이전트/채팅으로 코드·문서 수정 요청하기
- [ ] **Claude**(또는 Cursor 내 모델)에게 **설계 문서(`docs/`)를 맥락으로 주고** 구현 순서를 맞추기
- [ ] **Git**으로 변경 이력을 남기는 습관 (선택이지만 실무와 동일)

---

## Slack·봇 개념

- [ ] **Slack App**은 [api.slack.com/apps](https://api.slack.com/apps)에서 만들고, **Bot Token / Signing Secret**은 `.env`에만 둔다
- [ ] **슬래시 커맨드**(`/todo` 등)는 **코드만으로는 안 되고**, Slack 앱 설정에 **등록 + 재설치**가 필요하다
- [ ] **Socket Mode**로 로컬에서 **ngrok 없이** 이벤트를 받을 수 있다 (`SLACK_APP_TOKEN`)
- [ ] 채널 **멘션**과 **DM**은 Slack 이벤트 종류가 다르다 (`app_mention` vs `message.im` 등)

---

## 이 프로젝트 구조

- [ ] 진입점(`src/index.js`) → 앱 생성(`src/app.js`) → **핸들러**(`src/handlers/`)로 명령·이벤트 분리
- [ ] **데이터**는 `dataService` → SQLite 파일(`DB_PATH`) 경로 이해
- [ ] 로컬은 **한 PC에서 프로세스가 살아 있어야** 봇이 동작한다 (전원 OFF면 중단)

---

## 실무와의 거리 (강의용 한 줄)

- [ ] 실제 서비스는 **항상 켜진 서버 + (보통) 관리형 DB**로 배포한다 — 로컬 실습은 그 **축소판**이다.

## 스스로 점검 질문

1. `.env`에 무엇이 들어가고, Git에 올리면 안 되는 것은 무엇인가?  
2. `/todo`가 “유효한 명령이 아닙니다”로 나올 때 **어디를 먼저 보나?** (Slack 앱 설정 vs 코드)  
3. PC를 껐다 켰을 때 봇을 다시 켜려면 **무슨 명령**을 어디서 실행하나? → [`local-bot-restart.md`](./local-bot-restart.md) 참고
